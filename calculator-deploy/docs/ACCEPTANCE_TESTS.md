# 靈魂萬花筒計算器｜驗收清單

## A. 自動回歸測試

Cloudflare 每次組建都必須通過核心測試，失敗時禁止部署。

目前核心版本 2.2.0 包含 27 項，涵蓋：

- 國曆轉農曆。
- 1989/05/28、1989/12/28 與 2026/07/06 固定換算。
- 農曆 2026/12/01 跨至國曆 2027/01/08。
- 凌晨 12 點使用 24。
- 國曆年月日時分主數鏈。
- 農曆主命數 `37/10/1`。
- `37/10/1` 靈魂等級為 5級。
- 國／農曆流年、流月、流日。
- 貴人數與第一木馬。
- Bob 案例國／農曆皆尚未過生日。

## B. 靜態來源與版本驗證

Cloudflare 組建還必須檢查：

- `package.json` 與核心引擎使用相同核心版本。
- 頁面顯示包含相同核心版本；純介面修訂可加 `-rN` 後綴。
- Service Worker cache name 包含相同核心版本；純介面修訂可加 `-rN` 後綴。
- `index.html` 直接包含 `layout-fix.css`，不依賴建置時注入。
- `index.html` 直接使用合併後的 `summaryBirthdayStatus`。
- `index.html` 分別包含 `summarySolarBirthdayDate` 與 `summaryLunarBirthdayDate`。
- `script.js` 不再引用 `summarySolarStatus` 或 `summaryLunarStatus`。
- `script.js` 分別寫入本年國曆生日與本年農曆生日。
- `script.js` JavaScript 語法可解析。
- 所有必要靜態檔案均存在。
- 建置只允許產生 `lunar-data.js`，不得再改寫 HTML、JS、CSS 或 Service Worker。

## E. 固定人工案例三｜閏月

此案例尚待從正式對照資料選定並由使用者確認。

必須驗收：

- 國曆生日自動換算為正確閏月。
- 畫面保留原始閏月標記。
- 計算月使用原月份加 1。
- 若加 1 後超過 12，顯示「需人工確認」，不得自行推算。
- 國曆與農曆生日狀態分別判斷。
- 本年國曆生日與本年農曆生日分開顯示。

P0 完成前必須補上實際生日、查詢日期與完整預期結果。

## F. iPhone Safari 驗收

- 首頁、輸入卡片與所有結果卡片左右對齊。
- 日期與時間欄位可正常開啟 iOS 選擇器。
- 國曆生日變更後自動帶出農曆。
- 計算後快速結果與詳細表格更新。
- 國曆／農曆生日狀態維持同一格。
- 本年國曆生日與本年農曆生日分成兩個獨立欄位。
- 本年農曆生日欄位同時顯示農曆日期與國曆對應日。
- 寬表格只在表格容器內左右滑動，整頁不橫向漂移。
- 清除重算後輸入與結果回到空白狀態。
- 清除後生日狀態格恢復為「國曆：—／農曆：—」。
- 複製快速結果與完整結果可貼到備忘錄，且包含兩種本年生日日期。
- 頁首顯示正確核心版本與介面修訂號。
- 農曆資料或核心程式載入失敗時，頁首顯示明確錯誤，不可只呈現無反應畫面。

## G. PWA 與快取驗收

- Safari 重新整理可取得最新版本。
- 加入主畫面後可開啟。
- 介面修訂時 cache name 同步更新。
- 新版部署後不持續顯示舊介面。
- Service Worker 啟動後主動檢查更新。
- 離線時至少能載入最近一次完整快取的介面與資料。
- 不快取第三方來源請求。

## H. Cloudflare 驗收

- 生產分支：`calculator-deploy-setup`。
- 根目錄：`calculator-deploy`。
- 組建命令：`npm run build`。
- 部署命令：`npx wrangler deploy`。
- 組建紀錄顯示靜態來源檢查 11/11 通過。
- 組建紀錄顯示公式測試 27/27 通過。
- 靜態來源、版本或公式測試失敗時組建必須失敗，不得發布正式版本。
- 正式 Worker 維持 `soul-kaleidoscope-v1`，網址不更換。

## I. 發版規則

每次正式修改至少完成：

1. 修改正式來源檔，不使用建置腳本臨時改寫頁面。
2. 新增或更新對應驗收案例。
3. 公式或核心異動：同步更新 `package.json`、核心版本、頁面版號與 PWA cache name。
4. 純介面異動：核心版本可維持不變，頁面與 cache 使用 `-rN` 修訂後綴。
5. 執行 `npm run check` 或 `npm run build`。
6. Cloudflare 組建成功。
7. 以至少一組固定案例做手機人工驗收。
8. 記錄變更內容與已知限制。
9. 公式異動必須同步更新 `RULES_LOCK.md`。
