→ npm run test:cases

## D. 日月綻放完整鏈

公式：

```text
raw_total = 出生月份數值 + 出生日數值
chain = 對 raw_total 持續化簡至個位數，保留完整鏈
```

要求：

- 國曆與農曆分開計算。
- 不得先把月份、日期各自縮減後只保存最後主數。
- 不得將 `MMDD` 當成日期字串逐位加總。
- `dayMoon` 保存最後主數。
- `dayMoonChain` 保存完整鏈。
- 主頁、報告、複製結果與萬花圖核對優先顯示完整鏈。

固定案例：

- 國曆 1991/09/23：`9＋23＝32/5`。
- 農曆 1991/08/16：`8＋16＝24/6`。
- 國曆 1989/05/28：`5＋28＝33/6`。
- 農曆 1989/04/24：`4＋24＝28/10/1`。

## E. 核心固定案例

### 案例一｜1989/05/28

輸入：國曆 `1989/05/28`、時間 `15:17`、農曆 `1989/04/24`、查詢 `2026/07/05`。

預期：

- 查詢農曆 `2026/05/21`。
- 國曆：流年 `25/7`、位格 2、流月 `44/8`、流日 `37/10/1`。
- 農曆：流年 `20/2`、位格 2、流月 `38/11/2`、流日 `34/7`。
- 國曆五階段：`27/9、32/5、42/6、48/12/3、56/11/2`。
- 農曆五階段：`27/9、31/4、37/10/1、43/7、51/6`。
- 農曆主命數靈魂等級 `5級`。
- 國曆日月綻放 `33/6`；陰曆日月綻放 `28/10/1`。
- 國曆貴人 7、第一木馬 4；農曆貴人 7、第一木馬 2。

### 案例二｜1989/12/28

輸入：國曆 `1989/12/28`、時間 `00:00`、農曆 `1989/12/01`、查詢 `2026/07/06`。

預期：

- 查詢農曆 `2026/05/22`。
- 計算時數 24。
- 農曆 `2026/12/01` 對應國曆 `2027/01/08`。
- 國曆尚未過生日，流年 `22/4`、位格 1。
- 農曆尚未過生日，流年 `13/4`、位格 1。

### 案例三｜1991/09/23

輸入：國曆 `1991/09/23`、時間 `11:17`、農曆 `1991/08/16`、查詢 `2026/07/11`。

預期：

- 查詢農曆 `2026/05/27`。
- 本農曆年生日對應國曆 `2026/09/26`。
- 國曆：流年 `23/5`、位格 8、流月 `32/5`、流日 `31/4`。
- 農曆：流年 `24/6`、位格 8、流月 `32/5`、流日 `37/10/1`。
- 國曆五階段：`20/2、29/11/2、34/7、36/9、44/8`。
- 農曆五階段：`20/2、28/10/1、35/8、37/10/1、45/9`。
- 國曆日月綻放 `32/5`；陰曆日月綻放 `24/6`。
- 國曆貴人 3、第一木馬 4；農曆貴人 6、第一木馬 1。

### 正式閏月案例

P0 關閉前仍須補上國曆生日、農曆閏月日期、查詢日期、原始月、計算月與完整預期結果。

## F. Soul Profile

每次成功計算必須建立：

- `meta`、`subject`、`source`、`calendar`。
- `numerology.solar`、`numerology.lunar`。
- `numerology.solar.horse.dayMoonChain`。
- `numerology.lunar.horse.dayMoonChain`。
- `outputs.report`。
- `outputs.soulKaleidoscope`。

驗收：

- `Profile.validate(profile).ok === true`。
- 國／農曆 `soulStages` 各有五階段。
- `primaryNumber` 與日階段最後主數一致。
- 主頁、報告與萬花圖核對讀取同一份 Soul Profile。
- 報告頁與萬花圖核對不得重新呼叫核心公式。
- 清除重算後移除 `window.__SOUL_PROFILE__` 並清空衍生畫面。
- 真實個案資料不得寫入 console、網址、GitHub 或 Cloudflare 日誌。

## G. SNGL Number Data、Position Data 與年度解讀

- Number Data 完整包含 0～9。
- Position Data 完整包含 1～9。
- 報告引擎必須產生國／農曆主命數、流年、今年位格、流年 × 位格與雙曆年度總結。
- 缺少可靠農曆流年時不得把空值解讀為 0，並保留人工核對提示。

## J. 五分頁分析介面

結果區必須包含：

1. 總覽。
2. 國曆。
3. 農曆。
4. 年度能量。
5. 萬花圖核對。

驗收：

- 預設開啟總覽。
- 點擊頁籤只顯示對應內容。
- 鍵盤左右方向鍵可循環切換頁籤。
- 手機頁籤可橫向滑動，但整頁不產生水平捲動。
- 首頁「流年與位格」直接切至年度能量。
- 首頁「靈魂萬花圖」直接切至萬花圖核對。
- 清除重算後頁籤仍可使用，衍生內容恢復空狀態。

### 年度能量

計算後應顯示：

- 國曆今年位格。
- 農曆今年位格。
- 國曆流年 × 位格整合。
- 農曆流年 × 位格整合。
- 雙曆年度能量總結。

### 萬花圖核對

固定 11 個位置：

| 位置 | 欄位 | 1991/09/23 預期 |
|---|---|---:|
| 中心 | 主命數 | 7 |
| 上方 | 國曆日月綻放 | 32/5 |
| 上方副標 | 陰曆日月綻放 | 24/6 |
| 左上 | 木馬（二） | 3 |
| 右上 | 木馬（一） | 4 |
| 左下 | 木馬（三） | 1 |
| 右下 | 木馬（四） | 7 |
| 左側 | 陰曆貴人 | 6 |
| 右側 | 陽曆貴人 | 3 |
| 下方 | 流年 | 23/5 |
| 最外圈 | 今年位格 | 8 |

驗收：

- 表格資料來自 Soul Kaleidoscope Visual Model `1.0.1`。
- 結果寫入 `profile.outputs.soulKaleidoscope`。
- 可複製完整位置核對表。
- 今年位格只顯示最後主數。
- 左陰右陽及四個木馬位置不可互換。
- 此階段只建立核對資料，不宣稱已完成圖像生成。
