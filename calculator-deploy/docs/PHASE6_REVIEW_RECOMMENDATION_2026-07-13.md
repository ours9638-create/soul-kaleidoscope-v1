## 2. 建議修正文案

### CTX-002｜複合數與基本數字分層引用

建議把限制調整為：

> 複合數層保存來源、形成脈絡、完整鏈式與位置角色；未經獨立來源核對及人工核准的複合數意義必須維持 Pending。複合數不得複製或改寫 0～9 基本本體；未來如建立正式複合數資料庫，必須使用獨立版本、來源與審核流程。

### CTX-004｜日月綻放是位置角色

建議輸出欄位固定為：

```text
daySeatFinal
dayMoonFullChain
dayMoonFinal
calendarType
sourceMonth
sourceDay
baseNumberRef
```

限制：

- 日座數只取出生日期本身的縮減結果。
- 日月綻放取出生月加出生日的完整縮減鏈。
- 國曆與農曆分開保存。
- 日月綻放不等同主命數，也不覆蓋其他位置。

### CTX-011｜重複數字與專屬次數語句的優先序

建議增加：

```text
已核准的專屬先天數次數語句
> 通用重複權重語句
> 無正式內容的中性缺口提示
```

通用權重只能說明「該頻率在命盤中較突出」，不能自行產生新的性格、事件或療癒結論。

### CTX-014｜靈魂等級引用

建議增加正式測試約束：

```text
input chain: 37/10/1
expected soul level: 5級
```

完整鏈式中的所有後天過程數都必須依現行 Engine 規則納入判斷，不得只讀第一段 `37`。

## 3. 來源型態判定

### 可直接支持 Adopt 的來源

- 使用者已確認的木馬公式與期間。
- 可驗算公式表的純數值計算方式。
- Calculator Engine 2.2.1 已通過的回歸測試。
- 《數字課程》的日座數、日月綻放及雙曆示意。

### 只能支持 Candidate 的來源

- 《數字課程》手寫註記。
- `123 皆具` 等尚無優先序的條件組合。
- `有 4＋先天數 1` 的組合資料。
- 數字 0／9 特殊頻率分類。
- 舊聊天摘要與二次整理。

## 4. 本輪不建議解鎖

以下內容維持封鎖：

1. 缺數正式解讀。
2. 數字 4 的一般 1～3 次解讀。
3. 數字 0 的一般 1～3 次解讀。
4. 除已確認局部案例外的 4 次、5 次以上內容。
5. 條件型組合的正式優先序。
6. 81 組流年 × 位格正式文案。
7. 彩油、精油、儀式內容回寫核心數字規則。
