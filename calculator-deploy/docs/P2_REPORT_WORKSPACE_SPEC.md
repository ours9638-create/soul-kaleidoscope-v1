# P2｜個案報告工作區與流年位格解讀規格

版本：`1.1`  
目標 App：`2.6.x`  
狀態：主要程式完成，等待 CI、Cloudflare 與 iPhone 實機驗收

## 1. 目的

由同一份 Soul Profile 產生可切換、可編輯、可複製、可列印的個案報告，並加入國曆與農曆的流年位格解讀。

報告頁不得重新計算公式，只讀取計算器已建立的 Soul Profile 與 SNGL 報告輸出。

## 2. 正式名稱

所有介面、純文字報告、PDF、驗收文件與後續資料庫統一使用：

- 靈魂數字。
- 國曆日月綻放。
- 陰曆日月綻放。
- 流年位格解讀。

不得再以「生命數字」或「內頻」作為正式欄位名稱。

## 3. 報告模式

### 快速版

適合簡短交付與手機閱讀：

- 基本資料。
- 雙曆靈魂數字摘要。
- 靈魂數字頻率解讀。
- 流年位格解讀。
- 個案補充。

預設隱藏五階段與完整結構。

### 完整版

適合正式個案報告：

- 快速版全部內容。
- 國／農曆五階段靈魂數字。
- 貴人、日座、日月綻放與四個木馬。
- 完整成熟展現、失衡提醒與行動建議。

### 老師版

適合內部校對與授課：

- 完整版全部內容。
- SNGL 代碼。
- role。
- 幾何原型。
- 數字配色。
- Number Data、Position Data 與 Report View 版本。

## 4. 報告段落

可個別顯示或隱藏：

1. 基本資料。
2. 靈魂數字摘要。
3. 靈魂數字解讀。
4. 流年位格解讀。
5. 五階段靈魂數字。
6. 靈魂數字結構。
7. 個案補充。

段落設定與草稿保存在該報告預覽的 localStorage payload 中。

## 5. 流年位格解讀

### 資料來源

- 流年主數：`data/sngl/numbers.v1.json`
- 今年位格：`data/sngl/positions.v1.json`
- 組裝邏輯：`public/sngl-report.js`

### 位格 1～9

| 位格 | 運作階段 |
|---:|---|
| 1 | 啟動與定位 |
| 2 | 協調與連結 |
| 3 | 表達與擴散 |
| 4 | 建構與落地 |
| 5 | 轉換與試驗 |
| 6 | 整合與承接 |
| 7 | 沉澱與校準 |
| 8 | 顯化與整合資源 |
| 9 | 完成與釋放 |

### 報告輸出

1. 國曆今年位格。
2. 農曆今年位格。
3. 國曆流年 × 位格整合。
4. 農曆流年 × 位格整合。
5. 雙曆年度能量總結。

### 雙曆年度總結原則

- 國曆流年作為外在事件與行動節奏的參考。
- 農曆流年作為內在感受與整合節奏的參考。
- 兩組位格相同時，指出共同運作階段。
- 兩組位格不同時，指出外在推進與內在承受需要協調。
- 不使用命定、必然、保證發生等斷言。

## 6. 報告資料流

```text
輸入資料
→ Core Engine 2.2.0
→ Soul Profile 1.0.0
→ SNGL Report Engine 1.1.0
→ Report View Model 1.1.0
→ report.html
→ 複製純文字／列印 PDF
```

## 7. 預覽與隱私

- 報告網址只帶隨機識別碼。
- 姓名、生日與出生時間不得出現在網址。
- 報告 payload 只保存在目前瀏覽器。
- 預覽 24 小時後失效。
- 最多保留 10 份預覽。
- 可手動清除此報告與草稿。
- 報告頁使用 `noindex,nofollow,noarchive`。

## 8. 草稿

可編輯：

- 整體觀察。
- 當期重點。
- 補充建議。

規則：

- 輸入後 700ms 自動儲存。
- 可手動儲存。
- 可重設模式、段落與補充文字。
- 重設不刪除原計算結果。
- 儲存空間不足時顯示明確訊息。

## 9. 純文字輸出

純文字報告必須遵守目前：

- 報告模式。
- 段落開關。
- 個案補充。

標題包含：

- 雙曆靈魂數字摘要。
- 靈魂數字頻率解讀。
- 流年位格解讀。
- 五階段靈魂數字。
- 靈魂數字結構。

頁尾包含：

- Soul Profile。
- 計算引擎。
- SNGL Report。
- Number Data。
- Position Data。
- Report View。

## 10. PDF

第一版使用瀏覽器列印：

- 操作列與報告設定面板不列印。
- 隱藏段落不產生空白頁。
- 表格不得超出 A4 寬度。
- 段落標題與第一段內容避免被分頁切開。
- iPhone Safari 可透過列印預覽分享為 PDF。

後續評估固定 PDF 引擎，用於完全一致的頁碼、目錄與封面。
