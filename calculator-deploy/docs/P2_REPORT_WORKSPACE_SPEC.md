# P2｜個案報告工作區與流年位格解讀規格

版本：`1.2`  
目標 App：`2.7.x`  
狀態：主要程式完成，等待 GitHub CI、Cloudflare 與 iPhone 實機驗收

## 1. 目的

由同一份 Soul Profile 產生可切換、可編輯、可複製、可列印的個案報告，並加入國曆與農曆的流年位格解讀。

報告頁不得重新計算公式，只讀取計算器已建立的 Soul Profile 與 SNGL 報告輸出。

- App：`2.7.0`

- Number Data：`1.0.0`
- Position Data：`1.0.0`

## 3. 正式名稱

統一使用：

- 靈魂數字。
- 國曆日月綻放。
- 陰曆日月綻放。
- 流年位格解讀。

不得以「生命數字」或「內頻」作為正式欄位名稱。

## 4. 日月綻放顯示

- 核心公式：出生月份數值＋出生日數值。
- 保留完整持續化簡鏈。
- 國曆與農曆分開保存。
- 報告優先顯示 `dayMoonChain`，舊資料沒有該欄位時才回退至 `dayMoon`。

固定案例：

- 1991/09/23：國曆日月綻放 `32/5`。
- 1991/08/16：陰曆日月綻放 `24/6`。
- 1989/05/28：國曆日月綻放 `33/6`。
- 1989/04/24：陰曆日月綻放 `28/10/1`。

## 5. 報告模式

### 快速版

基本資料、雙曆摘要、靈魂數字頻率解讀、流年位格解讀與個案補充；預設隱藏五階段與完整結構。

### 完整版

快速版全部內容，加上雙曆五階段、貴人、日座、兩組日月綻放完整鏈與四個木馬。

### 老師版

完整版全部內容，加上 SNGL 代碼、role、幾何原型、數字配色與所有版本資訊。

## 6. 段落開關

1. 基本資料。
2. 靈魂數字摘要。
3. 靈魂數字解讀。
4. 流年位格解讀。
5. 五階段靈魂數字。
6. 靈魂數字結構。
7. 個案補充。

設定與草稿保存在該報告預覽的 localStorage payload。

## 7. 流年位格解讀

資料來源：

- 流年主數：`data/sngl/numbers.v1.json`。
- 今年位格：`data/sngl/positions.v1.json`。
- 組裝：`public/sngl-report.js`。

報告輸出：

1. 國曆今年位格。
2. 農曆今年位格。
3. 國曆流年 × 位格整合。
4. 農曆流年 × 位格整合。
5. 雙曆年度能量總結。

原則：國曆作為外在事件與行動節奏參考；農曆作為內在感受與整合節奏參考；不使用命定、必然或保證發生等斷言。

## 8. 資料流

```text
輸入資料
→ Core Engine 2.2.1
→ Soul Profile 1.0.0
→ SNGL Report Engine 1.1.0
→ Report View Model 1.1.1
→ report.html
→ 複製純文字／列印 PDF
```

## 9. 預覽與隱私

- 報告網址只帶隨機識別碼。
- 姓名、生日與出生時間不得出現在網址。
- payload 只保存在目前瀏覽器。
- 預覽 24 小時後失效，最多保留 10 份。
- 報告頁使用 `noindex,nofollow,noarchive`。

## 10. 草稿

可編輯整體觀察、當期重點與補充建議；輸入後 700ms 自動儲存，也可手動儲存或重設設定。

## 11. 純文字與 PDF

純文字必須遵守目前模式、段落開關與個案補充，並顯示正式名稱與完整日月綻放鏈。

第一版 PDF 使用瀏覽器列印：操作列與編輯器不列印，隱藏段落不產生空白頁，表格不得超出 A4。
