# 靈魂萬花筒｜產品與技術路線圖

## 一、產品定位

「靈魂萬花筒」是一套整合雙曆靈魂數字、出生時間、年度能量、個案管理、報告與靈魂萬花圖的個人分析系統。

目前產品型態：

- 手機優先 PWA。
- GitHub 自動測試與版本管理。
- Cloudflare Worker 固定網址部署。
- 個案資料預設保存在目前瀏覽器。
- 計算、資料模型、解讀、報告與圖像輸出分層維護。

## 二、核心原則

1. 網站、PWA、PDF、AI 解讀與靈魂萬花圖只讀取同一份 Soul Profile JSON。
2. App 版本與計算引擎版本分開管理；功能更新不得被誤認為公式更新。
3. 每次發布必須通過自動測試、靜態來源檢查、版本檢查與人工驗收。
4. 姓名、生日與出生時間不得寫入網址、GitHub、Cloudflare 日誌或公開測試資料。
5. 未核准的解讀內容不得進入客戶報告。
6. 核心解讀不使用 Aura-Soma 彩油內容；精油僅屬輔助建議，不影響數字計算。
7. 正式名稱統一為：靈魂數字、國曆日月綻放、陰曆日月綻放、流年位格解讀。
8. 「生命數字」與「內頻」不再作為正式欄位名稱。
9. 公開介面範例使用「個案 A」等中性名稱，不使用真實或既有個案名稱。

## 三、版本矩陣

| 模組 | 目前版本 | 說明 |
|---|---:|---|
| App／PWA | `2.7.0` | 紫金宇宙首頁、五分頁分析、年度能量、萬花圖核對與報告工作區 |
| 計算引擎 | `2.2.0` | 雙曆換算、五階段、流年、位格、流月、流日、貴人與木馬 |
| Soul Profile Schema | `1.0.0` | 網站、報告與圖像共同資料模型 |
| SNGL Report Engine | `1.1.0` | 主命數、流年、位格、流年 × 位格與雙曆年度總結 |
| SNGL Number Data | `1.0.0` | 0～9 基礎頻率內容 |
| SNGL Position Data | `1.0.0` | 今年位格 1～9 運作階段 |
| Report View Model | `1.1.0` | 快速版、完整版、老師版與年度段落 |
| Soul Kaleidoscope Visual Model | `1.0.0` | 固定 11 個位置的核對資料模型 |
| Case Store Module | `1.0.1` | LocalStorage、驗證、遷移、匯出與合併匯入 |
| Case Store Schema | `1` | 本機個案資料格式 |

### 版本規則

- 公式異動：提升計算引擎版本，更新 fixture、測試與 `RULES_LOCK.md`。
- App 功能異動：提升 App 版本；計算引擎可以維持不變。
- Soul Profile 異動：提升 Schema 並提供 migration。
- SNGL 內容異動：提升 Number Data 或 Position Data 版本。
- 報告組裝異動：提升 Report Engine 或 Report View Model。
- 萬花圖位置模型異動：提升 Soul Kaleidoscope Visual Model。
