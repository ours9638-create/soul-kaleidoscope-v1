# 靈魂萬花筒計算器｜產品與技術路線圖

## 核心原則

- 先鎖定公式，再開發資料保存與報告功能。
- 網站、PWA、PDF、AI 解讀與靈魂萬花圖只讀取同一份 Soul Profile JSON。
- 先做本機資料，再做雲端同步。
- 每一階段都必須有版本、驗收條件、回復方式與自動測試。
- 正式來源檔可直接閱讀，不使用建置腳本臨時改寫 HTML、JS、CSS 或快取版號。
- 未核准的解讀內容不得進入客戶報告。

## P0.5｜共同平台基礎

狀態：第一版已完成並接入正式網站

### A. 正式回歸測試

已建立：

- `tests/fixtures/regression-cases.json`
- `tests/run-regression-tests.js`
- `npm test`

目前固定案例：

1. `1989/05/28`：完整國／農曆五階段、流年、位格、流月、流日、貴人與木馬。
2. `1989/12/28`：凌晨 00:00 與農曆生日跨至下一國曆年。
3. `1991/09/23`：使用者正式案例，包含國／農曆主數、流年、位格、貴人與木馬。

Cloudflare 執行順序：

```text
npm test
→ build-calculator.js
→ 靜態來源與版本檢查
→ 27 項核心自檢
→ Wrangler 部署
```

### B. 統一 Soul Profile JSON

已建立：

- `schemas/soul-profile.schema.json`
- `public/profile-model.js`

模型版本：`1.0.0`

每次計算會產生：

- `meta`：模型、引擎、產生時間與個案 ID。
- `subject`：姓名。
- `source`：國曆生日、出生時間、查詢日期與農曆生日。
- `calendar`：查詢農曆與生日年度定位。
- `numerology.solar`：國曆五階段、主數、流年與木馬。
- `numerology.lunar`：農曆五階段、主數、流年與木馬。
- `outputs.report`：SNGL 報告資料。
- `outputs.soulKaleidoscope`：後續靈魂萬花圖資料。

目前網站已使用 Soul Profile 作為畫面與複製結果的共同資料來源，並暫時提供於 `window.__SOUL_PROFILE__` 供開發驗證。

### C. SNGL 與報告引擎

已建立：

- `data/sngl/numbers.v1.json`
- `public/sngl-report.js`
- build 時產生 `public/sngl-data.js`

版本：

- SNGL Report Engine：`1.0.0`
- 0～9 Number Data：`1.0.0`

第一版段落：

1. 國曆主命數。
2. 農曆主命數。
3. 國曆流年。
4. 農曆流年；缺少可驗算資料時不以 0 代替，改標記需人工確認。

每個段落已包含 SNGL 代碼、完整鏈條、頻率觀察、成熟展現、失衡提醒、行動建議、幾何原型、數字配色與客戶報告文字。

## P1｜個案本機儲存、搜尋與備份

目標版本：`2.3.x`  
狀態：資料模型與技術規格完成，可進入 UI 開發

### 功能

- 儲存目前個案到瀏覽器本機。
- 個案列表、姓名／生日搜尋與最近修改排序。
- 載入、覆寫、另存、刪除。
- JSON 匯出備份與 JSON 匯入還原。
- 匯入前預覽筆數、重複資料與格式錯誤。
- 每筆紀錄保存 schema、引擎、SNGL 與建立／修改時間。

### 資料原則

- 預設不送到伺服器。
- 不在網址參數、GitHub、Cloudflare 日誌或錯誤訊息中保存完整個案資料。
- 儲存原始輸入與版本；載入後由目前引擎重算。
- 每筆個案使用獨立 UUID，同名個案可並存。

### 完成門檻

- 可儲存至少 100 筆個案。
- 匯出、清空、重新匯入可完整還原。
- 舊 schema 可遷移。
- 錯誤匯入不破壞既有資料。

詳細規格：`P1_STORAGE_SPEC.md`

## P4｜靈魂萬花圖

目標版本：`3.2.x`

### 功能

- 從 Soul Profile 自動產生固定核對表。
- 判斷生日前版與生日後版。
- 套用固定九宮格、同心圓與 0～9 幾何原型。
- 輸出少字精準版提示詞。
- 今年位格圖面只顯示最後主數。

### 完成門檻

- 出圖前通過位置與數字核對。
- 外圈可變化，核心主頻不可改動。
- 配色只使用數字對應色，不使用彩油瓶顏色。
