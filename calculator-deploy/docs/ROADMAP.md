# 靈魂萬花筒｜產品與技術路線圖

## 一、產品定位

「靈魂萬花筒」是一套整合雙曆靈魂數字、出生時間、年度能量、個案管理、報告與靈魂萬花圖的個人分析系統。

目前產品型態為：

- 手機優先的 PWA。
- GitHub 自動測試與版本管理。
- Cloudflare Worker 固定網址部署。
- 個案資料預設保存在使用者目前瀏覽器。
- 計算、資料模型、解讀內容、報告與圖像輸出分層維護。

## 二、核心原則

1. 公式引擎、資料保存、解讀內容、報告與圖像輸出分層維護。
2. 網站、PWA、PDF、AI 解讀與靈魂萬花圖只讀取同一份 Soul Profile JSON。
3. App 版本與計算引擎版本分開管理；功能更新不得被誤認為公式更新。
4. 個案資料預設只保存在目前裝置；登入、加密與雲端同步延後至 P5。
5. 每次正式發布都必須通過自動測試、靜態來源檢查、版本檢查與人工驗收。
6. 姓名、生日與出生時間不得寫入網址、GitHub、Cloudflare 日誌或公開測試資料。
7. 未核准的解讀內容不得進入客戶報告。
8. 核心解讀不使用 Aura-Soma 彩油內容；精油僅屬輔助建議，不影響數字計算。
9. 正式名稱統一為：
   - 靈魂數字。
   - 國曆日月綻放。
   - 陰曆日月綻放。
   - 流年位格解讀。
10. 「生命數字」與「內頻」不再作為 App、報告與文件的正式欄位名稱。

## 三、版本矩陣

| 模組 | 目前版本 | 說明 |
|---|---:|---|
| App／PWA | `2.6.0` | 紫金宇宙品牌首頁、個案管理、報告工作區與流年位格解讀 |
| 計算引擎 | `2.2.0` | 雙曆換算、五階段、流年、位格、流月、流日、貴人與木馬 |
| Soul Profile Schema | `1.0.0` | 網站、報告與圖像的共同資料模型 |
| SNGL Report Engine | `1.1.0` | 主命數、流年、今年位格、流年 × 位格與雙曆年度總結 |
| SNGL Number Data | `1.0.0` | 0～9 基礎頻率內容 |
| SNGL Position Data | `1.0.0` | 今年位格 1～9 的運作階段解讀 |
| Report View Model | `1.1.0` | 快速版、完整版、老師版與流年位格段落 |
| Case Store Module | `1.0.1` | LocalStorage、驗證、遷移、匯出與合併匯入 |
| Case Store Schema | `1` | 本機個案資料格式 |

### 版本規則

- **公式異動**：提升計算引擎版本，更新 fixture、測試與 `RULES_LOCK.md`。
- **App 功能異動**：提升 App 版本；計算引擎可以維持不變。
- **Soul Profile 異動**：提升 Schema 版本並提供 migration。
- **SNGL 數字內容異動**：提升 Number Data 版本。
- **今年位格內容異動**：提升 Position Data 版本。
- **報告組裝邏輯異動**：提升 SNGL Report Engine 或 Report View Model 版本。
- **純視覺異動**：可提升 App 修訂版本，但不得改變公式或資料結構。

## 五、P0.5｜共同平台基礎

狀態：第一版完成並接入正式網站

### Regression Suite

- `tests/fixtures/regression-cases.json`
- `tests/run-regression-tests.js`
- `npm run test:core`
- 驗證核心 27 項自檢、正式 fixture、Soul Profile、SNGL 與流年位格輸出。

### Soul Profile JSON

- `schemas/soul-profile.schema.json`
- `public/profile-model.js`
- 每次計算建立 `meta`、`subject`、`source`、`calendar`、`numerology` 與 `outputs`。
- 報告與靈魂萬花圖不得自行建立另一套計算結果。

### SNGL 資料層

- `data/sngl/numbers.v1.json`
- `data/sngl/positions.v1.json`
- `public/sngl-report.js`
- Build 產生 `public/sngl-data.js`，同時載入 Number Data 與 Position Data。
- 缺少可靠農曆流年時不把空值解讀為 0。
