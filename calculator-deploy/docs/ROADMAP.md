# 靈魂萬花筒計算器｜產品與技術路線圖

## 一、核心原則

1. 公式引擎、資料保存、解讀內容、報告與圖像輸出分層維護。
2. 網站、PWA、PDF、AI 解讀與靈魂萬花圖只讀取同一份 Soul Profile JSON。
3. App 版本與計算引擎版本分開管理；功能更新不得被誤認為公式更新。
4. 先完成本機資料閉環，再評估登入、加密與雲端同步。
5. 每一階段都必須有自動測試、人工驗收、資料遷移與回復方式。
6. 個案姓名、生日與出生時間不得寫入網址、GitHub、Cloudflare 日誌或公開測試資料。
7. 未核准的解讀內容不得進入客戶報告。
8. 核心解讀不使用 Aura-Soma 彩油內容；精油僅屬輔助建議，不影響數字計算。

| 模組 | 目前版本 | 說明 |
|---|---:|---|
| App／PWA | `2.4.0` | P1 個案管理＋P2a 新分頁報告預覽 |
| 計算引擎 | `2.2.0` | 國／農曆、五階段、流年、位格、流月、流日、貴人、木馬 |
| Soul Profile Schema | `1.0.0` | 網站、報告與圖像的共同資料模型 |
| SNGL Report Engine | `1.0.0` | 第一版數字報告段落組裝 |
| SNGL 0～9 Data | `1.0.0` | 0～9 基礎頻率內容 |
| Case Store Module | `1.0.1` | LocalStorage、驗證、遷移、匯出與合併匯入 |
| Case Store Schema | `1` | 本機個案資料格式 |
| Report Preview Spec | `1.0` | 新分頁唯讀報告、列印與 24 小時預覽 |

### 版本規則

- **公式異動**：提升計算引擎版本，更新固定案例、測試與 `RULES_LOCK.md`。
- **App 功能異動**：提升 App 版本；計算引擎可維持不變。
- **資料模型異動**：提升 Schema 版本並提供 migration。
- **SNGL 內容異動**：提升 dataVersion，保留來源、核准狀態與變更紀錄。
- **純視覺異動**：可使用修訂後綴，但不得改變公式或資料結構。
