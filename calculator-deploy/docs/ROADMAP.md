# 靈魂萬花筒｜產品與技術路線圖

## 一、產品定位

「靈魂萬花筒」整合雙曆靈魂數字、出生時間、年度能量、個案管理、報告與靈魂萬花圖。

目前產品型態：手機優先 PWA、GitHub 自動測試、Cloudflare Worker 固定網址部署；個案資料預設只保存在目前瀏覽器。

## 二、核心原則

1. 網站、PWA、PDF、解讀與靈魂萬花圖只讀取同一份 Soul Profile。
2. App、計算引擎、資料模型、解讀資料、報告模型與圖像模型分開版本管理。
3. 姓名、生日與出生時間不得寫入網址、GitHub、Cloudflare 日誌或公開測試資料。
4. 正式名稱：靈魂數字、國曆日月綻放、陰曆日月綻放、流年位格解讀。
5. 公開介面使用「個案 A」等中性範例，不使用真實或既有個案名稱。
6. 日月綻放必須保存完整鏈；今年位格在靈魂萬花圖只顯示最後主數。
7. 核心解讀不使用 Aura-Soma 彩油內容；精油僅屬後續輔助建議。

### 版本規則

- 公式或核心輸出異動：提升引擎版本，更新 fixture、測試與 `RULES_LOCK.md`。
- App 功能異動：提升 App 版本。
- Soul Profile 異動：提升 Schema 並提供 migration。
- 解讀內容異動：提升對應 Data Version。
- 報告或萬花圖組裝異動：提升對應 Model Version。
