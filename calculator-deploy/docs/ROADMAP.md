# 靈魂萬花筒計算器｜產品與技術路線圖

## 核心原則

- 公式引擎、資料保存、報告與圖像輸出必須分層。
- 網站、PWA、PDF、AI 解讀與靈魂萬花圖只讀取同一份 Soul Profile JSON。
- 產品版本與計算引擎版本分開管理。
- 先完成本機資料流程，再評估帳號與雲端同步。
- 每一階段都要有自動測試、人工驗收、資料遷移與回復方式。
- 個案姓名、生日與出生時間不得寫入 GitHub、Cloudflare 日誌、網址參數或公開測試資料。
- 未核准的解讀內容不得進入客戶報告。

- 功能異動：提升 App 版本；計算引擎可維持不變。
- 純視覺修正：可使用修訂後綴，但不得改變資料結構或公式。

3. `1991/09/23`：使用者正式案例，包含國／農曆主數、流年、位格、貴人與木馬。

### B. Soul Profile JSON

已建立：

- `schemas/soul-profile.schema.json`
- `public/profile-model.js`

每次計算產生：

- `meta`：Schema、引擎、產生時間與個案 ID。
- `subject`：姓名。
- `source`：國曆生日、出生時間、查詢日期與農曆生日。
- `calendar`：查詢農曆與生日年度定位。
- `numerology.solar`：國曆五階段、主數、流年與木馬。
- `numerology.lunar`：農曆五階段、主數、流年與木馬。
- `outputs.report`：SNGL 報告資料。
- `outputs.soulKaleidoscope`：靈魂萬花圖預留欄位。

### C. SNGL 與報告引擎

已建立：

- `data/sngl/numbers.v1.json`
- `public/sngl-report.js`
- Build 時產生 `public/sngl-data.js`

第一版段落：

1. 國曆主命數。
2. 農曆主命數。
3. 國曆流年。
4. 農曆流年；資料不足時略過，不以空值當成數字 0。

### 完成門檻

- 產圖資料與報告資料來自同一 Soul Profile。
