# P2a｜新分頁報告預覽規格

版本：`1.0`  
目標 App：`2.4.0`  
狀態：第一版程式完成，等待 Cloudflare 與 iPhone 實機驗收

## 1. 使用流程

```text
完成有效計算
→ 點「在新分頁開啟報告」
→ 主頁保存暫時報告快照
→ report.html 在新分頁讀取快照
→ 顯示報告或列印／儲存 PDF
```

## 2. 資料原則

- 報告頁只讀取主頁產生的 Soul Profile，不重新計算。
- 網址只包含隨機識別碼，不包含姓名、生日、出生時間或計算結果。
- 暫存鍵：`soul-kaleidoscope.report-preview.<token>`。
- 報告預覽保存於同一瀏覽器的 LocalStorage。
- 預覽有效期 24 小時。
- 最多保留 10 份有效預覽；過期與超額資料自動清理。
- 使用者可在報告頁點「清除此報告」立即移除。

## 3. 報告內容

第一版顯示：

1. 個案姓名與產生時間。
2. 國曆生日、農曆生日、出生時間、查詢日期與查詢日農曆。
3. 國／農曆主命數、流年、今年位格、流月與流日。
4. 國曆主命數 SNGL 段落。
5. 農曆主命數 SNGL 段落。
6. 國曆流年 SNGL 段落。
7. 農曆流年 SNGL 段落；資料不足時不產生錯誤的 0 號解讀。
8. 國／農曆貴人數、日座、日月綻放與四個木馬。
9. Soul Profile、計算引擎、SNGL Report 與 SNGL Data 版本。

每個 SNGL 段落顯示：

- 完整數字鏈。
- 核心主題。
- 頻率觀察。
- 成熟展現。
- 失衡提醒。
- 行動建議。
- SNGL 代碼。
- 幾何與配色。

## 4. 操作

- 返回計算器。
- 列印／儲存 PDF。
- 清除此報告。
- 手機與桌面響應式版面。
- 列印時隱藏操作按鈕，使用 A4 版面並避免段落跨頁切斷。

## 5. 隱私

- 報告快照包含個案資料，僅保存於目前瀏覽器。
- 隨機識別碼不是個案資料，但取得同一裝置與完整網址的人可能在有效期內開啟報告。
- 共用裝置使用後應點「清除此報告」。
- 清除 Safari 網站資料會移除預覽與本機個案。
- 報告頁設定 `noindex,nofollow,noarchive`。
- 不將報告內容傳送至 GitHub、Cloudflare 日誌或第三方分析服務。

## 6. PWA

Service Worker 必須快取：

- `report.html`
- `report.css`
- `report.js`
- `report-preview.js`

已建立過報告預覽且資產已快取時，離線狀態應可重新開啟該報告。

## 7. 自動檢查

Cloudflare Build 必須驗證：

- 主頁包含 `openReportBtn`。
- 主頁載入 `report-preview.js`。
- 報告頁、CSS 與 JS 存在。
- 報告頁包含列印與清除操作。
- 報告預覽只讀取 `window.__SOUL_PROFILE__`。
- 報告預覽包含 24 小時失效資訊。
- 報告頁提供 `window.print()`。
- 報告頁可刪除本次預覽。
- Service Worker 快取全部報告資產。
- 所有報告 JavaScript 語法可解析。

## 8. 人工驗收

1. 未計算時按報告按鈕，顯示「請先完成計算」。
2. 計算後按按鈕，新分頁顯示正確姓名與數字。
3. 報告網址不包含姓名、生日或時間。
4. 報告重新整理後仍可顯示。
5. 主頁改算另一位個案後，舊報告分頁內容不被改寫。
6. 同時開啟兩份不同報告，內容互不覆蓋。
7. Safari 列印預覽排版完整。
8. 儲存 PDF 後無操作按鈕、無截字、無漏頁。
9. 點「清除此報告」後，重新整理不再顯示內容。
10. 24 小時過期預覽顯示重新產生提示。
11. 主畫面 PWA 可開啟新報告分頁。
12. 無網路時，已快取的報告頁與有效預覽仍可開啟。
