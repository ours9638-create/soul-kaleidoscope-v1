# Canva A4 頁面對應表

這份文件把系統輸出的 `CanvaPackage.pages` 對應到 Canva「靈魂萬花筒」專案資料夾中的 A4 報告頁面。

Canva 資料夾：`https://www.canva.com/folder/FAHNdHzaB5o`

## 定位

- Google Docs 是文字與版本主檔。
- Canva 是客戶最終視覺版，不是內容資料庫。
- A5 內部手卡不放進客戶報告。
- 7 月前採人工套版，不接 Canva API。

這樣做的風險是：如果 Canva 直接變成改稿主場，系統會失去版本追蹤。建議所有文字修改都回 Google Docs，再重新產出 CanvaPackage。

## A4 頁面對應

| 系統 pageType | Canva 頁名 | 版位 | 用途 | 套版注意 |
| --- | --- | --- | --- | --- |
| `cover` | 封面 | P01 | 個案名稱、報告名稱、版本識別 | 只放個案稱呼與報告標題，不放內部計算表。 |
| `opening` | 這次閱讀的焦點 | P02 | 放已核准摘要，建立報告與個案當下議題的連結 | 使用 `approvedClientSummary`；不要貼原始諮詢筆記。 |
| `core-facts` | 核心數字 | P03 | 列出國曆／農曆主命數、日月綻放（國曆日月數）、內頻（農曆日月數）、流年與位格 | 保留鏈條與最後主數；避免塞入完整內部手卡。 |
| `mature-expression` | 成熟運作方式 | P04 | 呈現個案可穩定運用的力量 | 挑 2–4 句重點，避免整段塞滿版面。 |
| `pressure-patterns` | 失衡與壓力模式 | P05 | 呈現陰影、過度或壓抑表現 | 好的與不好的都要說，避免只寫正向形容。 |
| `observable-signals` | 觸發與可觀察訊號 | P06 | 讓個案知道什麼狀態代表模式正在發生 | 用條列，讓個案能自我辨識。 |
| `relationship-expression` | 關係中的表現 | P07 | 整理關係、人際互動、界線與情緒表現 | 若該案沒有關係內容，這頁可省略或併入成熟／陰影頁。 |
| `work-expression` | 工作與行動表現 | P08 | 整理工作、決策、行動與資源承接方式 | 若該案工作議題不強，可改為生活實踐頁。 |
| `monthly-reminder` | 月份提醒 | P09 | 只放已核對的流月或月份提醒 | 未核對時整頁省略，不顯示待核對或猜測內容。 |
| `actions` | 接下來可以做的事 | P10 | 放 1–3 個可執行行動 | 每個行動要具體，可在一週內開始。 |
| `closing` | 閱讀提醒與結語 | P11 | 收束報告，保留非標籤化與非醫療宣稱邊界 | 保留支持語氣，不寫保證結果。 |

## 兩份報告差異檢查

若要確認兩份個案不是同一套模板硬套，先下載兩份 `CanvaPackage` JSON，執行：

```bash
npm run compare:canva -- path/to/case-a.json path/to/case-b.json
```

檢查重點：

- `caseId` 不可相同。
- `displayName` 若相同，要人工確認是否同一人重產。
- 文字重疊率超過 `0.55` 時，需要人工檢查是否模板感過重。
- 來源 `sourceContentIds` 重疊率超過 `0.55` 時，需要確認兩案是否真的有相近數字結構。
- 頁面順序相同不一定有問題；A4 報告本來會共享架構，真正要看的是每頁文字、重點句與行動建議是否不同。

這樣做的風險是：只用文字重疊率無法判斷設計美感，也無法看到 Canva 裡人工改過但沒回寫的文案。建議交付前仍要人工看一次 Canva 最終 PDF。
