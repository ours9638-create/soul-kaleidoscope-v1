# 美數社數字內容研究整理

資料來源：`G:/我的雲端硬碟/靈魂萬花筒/deep-research-report.md`

## 定位

這份研究只作為外部內容觀察，不併入靈魂萬花筒核心公式。

這樣做的風險是：如果把外部頻道的「號人」敘事、生日加總範例或節目文案直接放進我們的內容資料庫，會讓品牌來源、公式邏輯與版權邊界混在一起。建議只取「內容結構」與「資料欄位設計」作為參考，不採用對方文案。

## 最大 3 個風險

1. 資料覆蓋不足  
   報告主要依 Podcast 描述文字，不是完整字幕或完整影片逐字稿。若拿它做頻率統計，結果會偏向標題與簡介中出現的數字，不代表整集節目內容。

2. 品牌與公式混淆  
   美數社使用「號人」敘事與神仙數字語境；靈魂萬花筒目前使用主命數、日月綻放、內頻、木馬、貴人、流年與位格。兩者不能直接合併。

3. 文案權利與風格風險  
   研究中有節目標題、描述與上下文摘錄。這些可以用來理解市場溝通方式，但不能直接複製成報告內容或商品文案。

你有沒有想過：如果我們太早把外部頻道的敘事方式放進報告，短期看起來內容變多，長期會讓使用者分不清你的系統到底是獨立方法，還是別人的語言重組。

## 可借鏡項目

| 項目 | 可借鏡方式 | 不可直接使用原因 |
| --- | --- | --- |
| 集數式內容整理 | 可用於未來內容研究資料表：來源、集數、日期、主題、數字類型 | 不代表我們要做 Podcast 分析產品 |
| 數字類型分類 | 可參考分類欄位：序數、日期、百分比、算式結果、統計數字 | 分類可借鏡，原始語句不可照抄 |
| 上下文摘錄 | 可用於內部研究註記，幫助判斷數字出現情境 | 若放到公開報告，需重新改寫 |
| 頻率統計 | 可作為內容市場觀察：哪些數字主題較常被討論 | 不可拿來推導靈魂萬花筒公式權重 |
| 方法侷限說明 | 可學習它如何揭露資料來源限制 | 不能把不完整資料包裝成完整結論 |

## 建議新增的研究資料欄位

若未來要建立 `外部內容研究` 分頁，建議欄位如下：

| 欄位 | 說明 |
| --- | --- |
| researchId | 研究資料唯一 ID |
| sourceName | 來源名稱，例如美數社 |
| sourceType | YouTube、Podcast、Instagram、網站 |
| episodeOrPostId | 集數或貼文 ID |
| publishedDate | 發布日期 |
| sourceUrl | 原始連結 |
| numberText | 原文中的數字文字 |
| normalizedNumber | 標準化數字 |
| numberCategory | 序數、日期、百分比、算式、統計、其他 |
| contextSummary | 自己改寫的上下文摘要 |
| usablePattern | 可借鏡的內容結構 |
| restrictedUse | 不可直接使用的原因 |
| actionRecommendation | 觀察、追蹤、改寫、不要採用 |

## 是否影響目前系統

目前不影響。

| 系統區塊 | 判斷 |
| --- | --- |
| `src/core/numerology.js` | 不修改。外部內容研究不能改公式。 |
| `src/core/templates.js` | 不修改。研究沒有新的圖像位置規則。 |
| `src/core/report.js` | 暫不修改。可以未來用於報告語氣研究，但不能直接塞文案。 |
| `apps-script/Code.gs` | 不修改。研究資料不需要進入個案輸出流程。 |
| Google Sheets 內容資料庫 | 可新增「外部內容研究」分頁，但不與公式主檔混用。 |

## 建議後續流程

1. 若要繼續研究，先補完整資料來源：YouTube 影片清單、官方描述、字幕或逐字稿。
2. 每筆資料只保留短摘要與分類，不保存大量逐字內容。
3. 將外部研究與內部公式分開維護。
4. 若要把研究結果轉成自己的報告語氣，必須重新撰寫，不沿用對方標題句型。

## 結論

這份報告對「內容市場觀察」有用，對「公式定稿」沒有直接作用。

建議把它放在研究層，不放進主流程。等靈魂萬花筒完成真實個案送出測試後，再考慮是否建立 `外部內容研究` 表，讓未來彩油、圖卡、課程或社群內容有可追蹤的參考來源。
