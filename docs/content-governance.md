# 靈魂萬花筒內容治理規格

版本：content-governance-2026-06-29

來源：內部研究報告（PDF evidence；未經人工核准不得直接升級為正式內容）

機器可讀設定：`config/content-qa-rubric.json`

## 1. 核心原則

內容治理的目的不是讓文字變漂亮，而是確保每一段解讀都能被產生、被判斷、被限制、被更新。

每段內容都要通過四個基本問題：

1. 它有沒有保留使用者的主體性？
2. 它有沒有說清楚一個具體主題？
3. 它有沒有同時說成熟與失衡，而不是只講好聽話？
4. 它有沒有讓使用者知道下一步？

如果答案是否定，這段文字不能進正式報告。

## 2. 內容資料結構

正式解讀內容仍以 `InterpretationBlock` 為主。最低欄位要求：

| 欄位 | 用途 |
|---|---|
| `contentId` | 穩定追蹤與版本比對 |
| `domain` | 主命數、日月綻放、內頻、組成數、靈魂等級、木馬、貴人、流年、位格 |
| `matchKey` | 數字、級數、鏈條或組合 |
| `polarity` | 成熟、陰影、平衡 |
| `context` | core、relationship、work、emotion、decision、monthly |
| `clientText` | 可進客戶報告文字 |
| `practitionerNote` | 只給解讀者看的提醒 |
| `trigger` | 觸發情境 |
| `observableSignal` | 客戶可辨識訊號 |
| `actionSuggestion` | 可執行行動 |
| `sourceType` | 來源類型 |
| `sourceRef` | 來源位置 |
| `sourcePage` | PDF 或文件頁碼 |
| `reviewStatus` | 待辨識、待核對、已核對、停用 |
| `contentVersion` | 內容版本 |
| `lastReviewedAt` | 最後核對日期 |

## 3. 內容產生順序

不要先寫長文。正確順序是：

1. 數字結構表：核心、資源、需求、失衡、成熟、誤解、情境、行動、提問、禁語。
2. 短句變體：一句解讀、提醒句、行動句。
3. 報告段落：依個案觀察與數字結果組合。
4. Canva 文字：從 Google Docs 主檔萃取短版，不另起一套內容。
5. 社群或課程文案：從已核對內容延伸，不反向污染正式資料庫。

這樣做的風險是前期會慢一點；建議接受，因為先寫長文會讓後期重寫成本更高。

## 4. 內容評分表

每段正式客戶文字至少 7 分才可進入 `reviewed` 或 `delivered`。

| 維度 | 0 分 | 1 分 |
|---|---|---|
| 主體性 | 替客戶下定義 | 保留「可觀察、可核對、可選擇」 |
| 非命定 | 有命定、絕對語氣 | 用支持、提醒、可能性語氣 |
| 主題清楚 | 只有形容詞 | 有明確主題 |
| 成熟與陰影 | 只說正面 | 成熟與失衡都有 |
| 情境相關 | 誰都能套 | 接到關係、工作、情緒、決策或月份 |
| 可行動 | 只有感受 | 有具體小步驟 |
| 不製造焦慮 | 放大恐懼或羞愧 | 語氣穩定、可承接 |
| 不確定性 | 寫成絕對答案 | 說明這是觀察入口 |
| 讀完知道下一步 | 沒有下一步 | 有今天／這週／這個月方向 |

下一批 Google Sheet 人工核對表必須加入四個欄位：`QA分數`、`QA阻塞原因`、`修改優先級`、`核對決策`。欄位細節見 `docs/review-sheet-qa-fields.md`。

## 5. 微行動模板

每個 `actionSuggestion` 優先轉成這三段：

- 今天：一個 10 分鐘內能做的小動作。
- 這週：一個可觀察的模式或訊號。
- 這個月：一個要避免的過度反應或消耗。

範例：

> 今天先寫下你正想開始的三件事，圈出其中不是為了被看見的那一件。這週觀察自己在哪些時刻會因怕落後而急著答應。這個月避免用過度承諾證明自己。

## 6. 詞庫治理

### 禁用詞庫

- 一定、絕對、注定、命中注定、使命就是。
- 宇宙要你、上天安排你、不可改變。
- 治療、治癒、療癒創傷、改善焦慮、改善憂鬱。
- 你就是這種人、你不能、你必須。

### 替代詞庫

| 禁用語 | 替代說法 |
|---|---|
| 你一定會 | 你可以先觀察是否容易 |
| 你命中注定 | 這組數字像是在提醒 |
| 你的使命就是 | 你可能正在練習的方向是 |
| 療癒你的創傷 | 支持你整理目前的感受與模式 |
| 你必須改變 | 你可以先從一個小步驟開始 |

### 情境詞庫

同一個概念要依情境轉譯。

| 概念 | 關係 | 工作 | 情緒 | 決策 |
|---|---|---|---|---|
| 界線 | 說清楚需求與可承接範圍 | 明確分工與責任邊界 | 辨認哪些情緒不是你的 | 設定決策條件 |
| 行動 | 主動開啟對話 | 推進一個可驗證步驟 | 先照顧身體與節奏 | 做最小可逆決定 |
| 整合 | 看見彼此立場 | 回收經驗並調整流程 | 承認感受再放下 | 比較選項與代價 |

## 7. 交互詮釋規則

避免把數字段落直接拼貼。至少先判斷四種交互：

| 類型 | 判斷方式 | 寫法 |
|---|---|---|
| 衝突型 | 兩個數字拉向不同需求 | 說明拉扯與可調和點 |
| 補強型 | 兩個數字支持同一方向 | 強化主題，但避免重複句 |
| 放大型 | 0、9 或重複數字讓主題變強 | 說明能量被放大後的成熟與失衡 |
| 延遲型 | 內在需求與外在展現不同步 | 說明為何知道但做不到，給小步驟 |

## 9. 導入順序

1. 先把本文件作為人工核對標準。
2. 下一批 Google Sheet 核對表加入「QA 分數」與「需修改原因」。
3. 報告生成器先不自動改寫，只依已核對內容組合。
4. 等 3–5 份真實個案回饋後，再決定是否讓 AI 做語氣微調。
