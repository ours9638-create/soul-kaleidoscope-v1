# 危機轉介模式實作規格

版本：crisis-referral-mode-2026-06-29

來源設定：`config/crisis-referral-policy.json`

## 1. 核心規則

只要使用者輸入碰到危機或專業範圍風險，系統必須停止數字解讀。

這個方向有一個明確取捨：會讓某些對話不能繼續用原本的溫柔數字語氣承接。建議接受，因為安全邊界比完整解讀重要。

## 2. 觸發條件

第一批先處理四類：

| 類型 | 等級 | 系統動作 |
|---|---|---|
| 自傷或自殺 | P0 | 停止數字解讀，提供緊急資源 |
| 立即危險或暴力威脅 | P0 | 停止數字解讀，建議 110／119 |
| 現實感混亂 | P1 | 停止數字解讀，建議真人專業支持 |
| 醫療、法律、重大財務決策 | P1 | 說明範圍限制，轉向專業人士 |

## 3. 回覆模板

預設繁中模板：

> 我先不做數字解讀，因為你現在提到的內容可能牽涉安全或專業協助。
>
> 如果你或身邊的人有立即危險，請先打 110 或 119。
>
> 如果你有想傷害自己、想死、或覺得撐不下去，請立刻聯絡 1925 安心專線、1995 生命線或 1980 張老師。
>
> 現在最重要的不是分析數字，而是先讓你有人可以陪、也讓安全被接住。

## 4. 報告生成限制

危機轉介模式觸發時：

- 不產生數字解讀。
- 不產生月提醒。
- 不產生「你目前的課題」這類詮釋語句。
- 不輸出命定、診斷、醫療、法律、財務建議。
- 可以輸出安全資源與「請找真人協助」的提醒。

## 5. 後續實作點

第一階段只做文件與設定。

第二階段可接：

- Web 表單送出前的文字掃描。
- Apps Script `save-and-generate-report` 前的風險檢查。
- Google Docs 報告產生器的阻擋狀態。
- `reviewIssues` 加入 `crisisReferralTriggered`。
