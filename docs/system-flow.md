# 完整流程

## v1 主流程：先選服務

1. 先選服務：數字盤、精油產品、數字盤 + 精油搭配。
2. 若是數字盤或組合服務，先計算固定主頻率與年度變化。
3. 若需要出圖，系統產出出圖核對表與 SVG 校對版。
4. 若包含精油，系統產出精油產品建議與安全提醒。
5. 系統產出對應服務的 Markdown 報告草稿。
6. 你確認數字、位置、精油語氣與安全提醒。
7. 若要美術版，使用報告中的規則與提示詞進一步生圖。
8. 報告與圖像放到 Google Drive，輸出紀錄寫回 Sheets。
9. 客人只拿到只讀連結，不進後台。

## 服務分流

### 靈魂萬花筒數字盤

數字盤是主服務。它負責排盤、出圖核對、SVG 校對版與報告草稿。

### 精油產品/配方

精油可以單獨作為一項產品或服務，不需要先排生日盤。它輸出產品建議、使用情境、安全提醒與配方草稿。

### 數字盤 + 精油搭配

組合服務一定先完成數字盤，再接精油支持建議。精油只作為支持層，不反推數字公式。

## 未來其他出圖組合

不要直接複製整套靈魂萬花圖邏輯。新增模板時照這個順序：

1. 在 `src/core/templates.js` 新增 template registry 項目。
2. 定義 `requiredFields`，說清楚該模板需要哪些計算結果。
3. 定義 `slots`，鎖定圖面位置。
4. 新增 `build...Checklist`，把計算結果映射到位置。
5. 新增測試，確認位置不會被重排。
6. 若需要不同 SVG 版型，再擴充 `src/core/svg.js`。

## 建議優先順序

1. 先穩定 `soul-kaleidoscope`。
2. 再做 `minimal-number-card` 這種低風險圖卡。
3. 最後才做美術型、商品型、手機桌布型模板。

這樣做的風險是初期視覺不夠華麗。建議先接受，因為校對版的責任是準，不是美。
