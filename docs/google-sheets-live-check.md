# Google Sheets 開工讀取範圍

這份文件定義「開工時要怎麼讀試算表」。重點是省用量：先用本機檔案掃描判斷有沒有更新，再依 `config/google-sheets-registry.json` 決定要不要用 Google Drive / Google Sheets 讀線上內容。

`config/google-sheets-registry.json` 是目前的試算表登錄表。未來你新增公式表、內容表、精油表或圖像規則表時，先更新登錄表，再讓開工流程引用同一份清單。

## 開工讀取規則

1. 先跑 `npm run work:start`。
2. 如果 `.workflow/start-report.md` 顯示試算表沒有更新，不主動讀線上表格。
3. 如果試算表有更新，先讀 `config/google-sheets-registry.json` 裡列出的表格 metadata，不直接全表讀取。
4. 只讀登錄表中跟當前任務有關的重點分頁與小範圍。
5. 讀到公式或欄位變更時，才修改程式或文件。

## 需要特別確認的衝突

- `02_公式定稿` 更新：可能影響核心計算。
- `09_欄位字典` 更新：可能影響 Sheets 欄位、Apps Script、報告輸出。
- `25_精油數字對應0-9` 到 `48_商品化檢查清單` 更新：可能影響精油模組，但不得反推數字公式。
- `59_數字生成幾何圖規則` 到 `66_圖像輸出檔案清單` 更新：可能影響 SVG 或生圖提示詞。
- `87_靈魂萬花圖欄位映射` 更新：可能影響圖像位置，必須重新跑圖像測試。

這樣做的風險是：開工會多一個判斷步驟。建議保留，因為試算表會持續更新，如果不先判斷更新類型，很容易把文案更新誤當公式更新。
