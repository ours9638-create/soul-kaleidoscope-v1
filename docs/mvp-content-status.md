- 65 頁課程 PDF 全頁影像索引；第一輪標記 50 頁疑似有手寫或課堂註記。
- 4 份試作個案完成結構與隱私檢查。
- 主命數 1–9 已在 Google 試算表完成核准，36 則成熟、陰影、關係與工作內容已升級為 `verified`；原 38 筆鏈條內容維持 `mainDestinyChain`，不與最終主命數混用。

- 靈魂等級 1–7 已在 Google 試算表完成核准，7 則客戶解讀已升級為 `verified`；宗教／神靈手寫補充維持排除，計算條件仍沿用已確認的 A/B/C 與 B/C 分階段規則。
- 日座數已依正式公式表接入快速閱讀與 A5 內部手卡：國曆／農曆分別取出生日最終數；日月數取月日相加後最終數。PDF 第 32 頁／印刷頁 34 只作欄位定義與頁碼證據，手寫補充未核准前不進客戶文案。
- 日座數 1–9 解讀草稿已建立，並新增至 Google 試算表 `日月綻放_內頻1-9_人工核對表_2026-06-21` 的 `日座數1-9` 分頁；目前狀態仍為 `待核對`。
- 日月綻放／內頻升級腳本已接好 approval manifest；目前雲端核對表 18 筆仍為 `待核對`，因此不建立 `verified` 內容。
- 貴人、流年、位格 1–9 已建立本地人工核對批次與 CSV 匯入包；目前尚未上傳為 Google Sheet，狀態維持 `pending-review`。
- 已新增 `npm run promote:reviewed`，用 Node.js 依 approval manifest 升級已核准內容，不再需要每次靠臨時 openpyxl fallback。

## 尚未達成交付條件

- 公式狀態仍是 `needs-live-recheck`：流年位格、木馬、日月綻放、內頻、日座數、流年需對正式公式表完成線上重核。
- 正式內容庫仍是 142 則，其中 52 則核對完成（主命數 36 則、木馬 9 則、靈魂等級 7 則），其餘 90 則維持 `pending-review`；另新增 13 則日月綻放組成數與 13 則內頻組成數核對草稿，未核准前不進客戶正式報告，也不計入正式內容庫。
- 4 份試作已有主命數、木馬與靈魂等級領域覆蓋，且包含關係與工作脈絡；日月綻放、內頻、貴人、流年與位格仍缺正式內容。
- PDF 的 50 頁疑似手寫頁面只是建立索引；目前只有影像頁 31–36 做第一批人工辨識，且仍為 `pending-review`。

因此目前系統可以產生安全草稿，但不得宣稱「7 月內容 MVP 已完成」，也不得把數字盤輸出升級為 `reviewed`。系統已用 `formulaStatus`、`contentCoverageStatus` 與 `reviewIssues` 自動阻擋誤交付。

## 內容升級指令

已核准內容要升級進正式資料時，先建立對應 `*-approval.json`，再執行：

```bash
npm run promote:reviewed
```

這個指令只會升級已有 approval manifest 的批次；沒有 approval 的日月綻放、內頻、貴人、流年、位格會維持 `pending-review`，不會被誤升級。
