# 服務模組模型

## 先講結論

採用「主系統 + 模組」。

- 主系統：靈魂萬花筒數字盤。
- 第一個正式模組：精油產品/配方。
- 組合服務：數字盤 + 精油搭配。
- 未來模組：彩油、圖卡、課程、年度報告、其他元素。

這個方向比較穩，因為品牌主軸不會散，精油也不會被降級成數字盤裡的一小段文字。

## 服務目錄

| 服務 ID | 名稱 | 是否可單獨使用 | 定位 |
|---|---|---:|---|
| `soul-number-reading` | 靈魂萬花筒數字盤 | 是 | 主服務 |
| `essential-oil-product` | 精油產品/配方 | 是 | 第一個產品模組 |
| `soul-number-with-oil` | 數字盤 + 精油搭配 | 否 | 組合服務 |

## 服務邏輯

### 靈魂萬花筒數字盤

用途：

- 個案排盤。
- 靈魂萬花圖校對版。
- 報告草稿。
- 未來美術版出圖提示詞。

輸入：

- 姓名/代稱。
- 國曆生日。
- 農曆生日。
- 出生時間。
- 查詢日期。

輸出：

- 主命數、日月綻放、內頻、木馬、貴人、流年、今年位格。
- SVG 校對版。
- Markdown 報告草稿。

### 精油產品/配方

用途：

- 可單獨販售或交付。
- 可當成數字盤後的支持建議。
- 可延伸成標籤、成本、庫存、批次紀錄。

輸入：

- 姓名/產品名。
- 使用情境。
- 產品型態。
- 建議精油。

輸出：

- 精油產品建議。
- 使用情境。
- 安全提醒。
- 產品/配方草稿。

安全邊界：

- 不寫治療。
- 不寫治癒。
- 不寫保證改善。
- 需保留孕期、嬰幼兒、慢性病、用藥、寵物等禁忌提醒。

### 數字盤 + 精油搭配

流程：

1. 先完成數字盤。
2. 產出出圖核對表。
3. 再建立精油支持建議。
4. 產出組合報告。

限制：

- 精油只作為支持層。
- 精油不得反推數字公式。
- 數字盤公式仍以公式主檔為最高來源。

## 未來新增模組規則

每個新模組都要先回答：

1. 它能不能單獨成為服務？
2. 它能搭配哪些服務？
3. 它需要哪些輸入？
4. 它輸出什麼？
5. 它有哪些安全或語氣限制？
6. 它會不會污染數字盤公式層？

如果第 6 題答案是會，這個模組不能直接接進主系統。

## 建議下一個模組

不要急著開很多模組。建議順序：

1. 先穩定 `soul-number-reading`。
2. 同步建立 `essential-oil-product` 的產品欄位。
3. 再做 `soul-number-with-oil` 的組合交付。
4. 等交付 5-10 份後，再新增彩油或圖卡。
